// This uikeys.txt was cobbled together by resopmok.
// This file meant to be used in conjunction with the Factory UI Keys widget by badosu
// Please find it here: https://gist.github.com/badosu/c656fa654aadc61b99df8db5e6b3c15f#file-factory_ui_keys-lua and
// drop it into the directory Beyond-All-Reason/data/LuaUI/Widgets/ (create this directory if it does not exist).
// Make sure to enable the widget from the f11 list after installing it.
// I also recommend using the badmenu widget (also by badosu), which adds the hotkey to the lab production menu and 
// redraws build menus only with cycle options. It is installed the same was as the Factory UI Keys Widget.
// Find it here: https://gist.github.com/badosu/fbba5a23c33300d56df2b8e87db4e819
// This uikeys was tested only on a QWERTY keyboard. Significant changes should be made for other keyboard layouts.
// Please note some differences with the current default hotkey layout:
// -> Production hotkeys added for factories using z -> / and h, j including Shift+ (add 5), Alt+ (insert as next
//    unit in queue), and Ctrl+ (dequeue)functionality.
// -> Clear factory queue added to e (works when a lab is selected).
// -> Factory repeat on/off added to r (works when a lab is selected).
// -> Select commander has moved to q from Ctrl+c.
// -> Select idle worker has moved to Ctrl+s from Ctrl+b.
// -> Custom army selections (select all of same type unit on map) have moved to ctrl+q through ctrl+i and ctrl+f.
// -> Ping the map removed from q, now only on `.
// -> Resurrect (rez bots, subs) and capture (commanders) added to t (when they are selected).
// -> Some debugs moved to Alt+o and Alt+p.
// -> Build keys for constructors have had some changes and additions, for example T1 energy converters added to z
// and T2 factories added to v.
// -> Mines are now on z, x, and c - after buildings, before units.
